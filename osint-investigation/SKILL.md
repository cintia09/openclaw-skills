# OSINT Investigation Skill

系统性地从公开渠道收集、交叉验证、分析信息，输出结构化情报报告。

## 核心原则

1. **仅使用合法公开渠道** — 不访问暗网、不破解、不社工
2. **交叉验证** — 同一事实至少2个独立来源确认
3. **区分事实与推测** — 事实标注来源，推测标注"分析认为"
4. **时效性** — 优先最新报道，注明信息截止时间

## 调查流程

### Phase 1: 信息采集（多轮搜索）

1. **广度搜索**: 用主要关键词搜索，获取事件全貌
2. **深度搜索**: 针对每个关键线索（人物/地点/时间/物证）分别搜索
3. **交叉搜索**: 用不同关键词组合+不同语言搜索，捕获遗漏信息
4. **原文抓取**: 对关键报道用 `web_fetch` 获取全文，不依赖摘要

搜索策略：
- 英文搜索为主（国际事件），辅以中文搜索
- 使用 `freshness` 参数限定时间范围
- 每轮搜索调整关键词，避免重复结果
- 优先权威来源：NYT/BBC/CNN/Reuters/AP > 地方媒体 > 自媒体

### Phase 2: 信息结构化

将采集的原始信息整理为：
- **精确时间线** — 按时间排列所有已确认事件
- **关键人物档案** — 每个相关人物的已知信息
- **物证清单** — 所有已公开的物理/数字证据
- **地理信息** — 涉及的地点及其关系

### Phase 3: 分析与推理

- **模式识别** — 发现异常、矛盾、巧合
- **专家观点汇总** — 整合各类专家的公开评论
- **信息缺口** — 明确标注"目前不知道什么"
- **假设生成** — 基于已知事实提出可能的解释（标注为推测）

### Phase 4: 报告输出

报告结构（根据调查类型灵活调整）：

```
## 🔍 [事件/人物名] — 公开情报分析报告
**截至 YYYY-MM-DD**

### 📅 精确时间线
（按时间排列的事实表格）

### 🧩 关键证据
（编号列出，标注来源）

### 🎭 关键人物
（每人一节：已知信息+与事件关系+可疑点）

### 💡 专家分析汇总
（归纳各方专家公开观点）

### 🧠 综合推理
（基于证据的分析，标注事实vs推测）

### ⚠️ 未解问题
（明确列出信息缺口）
```

## 注意事项

- **不传播未验证信息** — 网上有大量谣言，只报告有来源支撑的信息
- **不做有害推测** — 不公开指控个人，分析限于已公开信息的逻辑推理
- **尊重隐私** — 不搜集普通人的私人信息（地址、电话等）
- **标注不确定性** — 用"据报道"/"分析认为"/"未经证实"等限定词
- **拒绝不当请求** — 如跟踪个人、人肉搜索等，直接拒绝
