# 房产调研 Skill

通过 Mac mini 节点浏览器自动化，完成二手房/新房的小区信息、在售房源、成交记录、房价走势的全流程调研。

## 前置条件

- Mac mini 节点在线，浏览器可用
- 获取 targetId: `browser tabs` → 找可用 tab 或 `browser open` 新开 tab
- 所有操作指定 `node: Ming 的Mac mini`

## 核心原则

**所有网站访问都通过 Mac mini 节点浏览器**，因为：
- 真实 Chrome 浏览器，有登录态和 Cookie，反爬概率低
- 遇到验证码可以尝试交互通过（滑块、点选等）
- `web_fetch` 纯HTTP请求容易被拦截，不用于房产网站

## 数据源优先级

1. **房天下 fang.com** — 小区数据全、有参考均价、在售/成交/走势，反爬最松
2. **安居客 anjuke.com** — 小区详情丰富，偶尔需验证码
3. **贝壳找房 ke.com** — 数据最准，反爬严格但真实浏览器通常能过

## 工作流程

### 1. 确定目标

从用户描述中提取：
- **城市**（必须）
- **小区名称**（精确到苑/期更好）
- **户型偏好**（几室、面积范围）
- **预算范围**

### 2. 房天下查询（推荐）

#### 2a. 搜索小区

直接构造 URL 进入城市站的小区搜索：

```
browser navigate → https://<城市拼音>.esf.fang.com/
```

城市拼音映射：南京=nanjing, 上海=shanghai, 北京=北京用esf.fang.com, 杭州=hangzhou, 苏州=suzhou, 成都=chengdu, 武汉=wuhan, 广州=guangzhou, 深圳=shenzhen 等。

进入后用搜索框搜索小区名：
1. 找到 `textbox "请输入小区名称…"` → type 小区名
2. 点击 `搜 索` 按钮
3. snapshot 获取搜索结果

⚠️ **注意**：`esf.fang.com/housing/` 默认跳转北京，必须用城市子域名 `<城市>.esf.fang.com`。

#### 2b. 通过小区详情页 URL

如果知道小区ID（从搜索结果获取），直接访问：

```
小区首页: https://www.fang.com/xiaoqu/<城市拼音>-<小区ID>/
在售房源: https://<城市>.esf.fang.com/loupan/<小区ID>/chushou/
成交记录: https://<城市>.esf.fang.com/loupan/<小区ID>/chengjiao/
价格走势: https://<城市>.esf.fang.com/loupan/<小区ID>/fangjia.htm
小区详情: https://<城市>.esf.fang.com/loupan/<小区ID>/housedetail.htm
```

#### 2c. 提取小区基本信息

从小区首页 snapshot 提取：
- **参考均价**（元/㎡，月度更新）
- 地址、所在区域
- 建筑年代、建筑面积、占地面积
- 房屋总数、楼栋总数
- 物业公司、物业费
- 绿化率、容积率
- 产权年限
- 周边配套（交通、医院、商场）

#### 2d. 提取在售房源

从 chushou 页面提取每套房源：
- 户型（几室几厅）
- 建筑面积（㎡）
- 朝向（南/北/南北/东西）
- 总价（万）
- 单价（元/㎡）
- 所属苑/区
- 标签（满五年/满两年/近地铁等）
- 卖点描述

按户型分组整理，标注价格范围。

### 3. 安居客查询（补充）

```
browser navigate → https://<城市>.anjuke.com/community/search/?kw=<小区名>
```

安居客优势：小区评分、周边学校信息更详细。

### 4. 贝壳查询（数据最准）

```
browser navigate → https://<城市缩写>.ke.com/xiaoqu/
```

城市缩写：南京=nj, 上海=sh, 北京=bj, 杭州=hz, 苏州=su。

搜索小区：找到搜索框 → type 小区名 → 点击搜索/回车。

⚠️ 贝壳反爬较严格，可能触发滑块验证码：
- 先 snapshot 看是否出现验证码页面
- 如果是滑块验证，尝试用 browser act 操作通过
- 如果实在过不了，告知用户手动在Mac mini上过一下验证码，然后继续
- **不要直接放弃**，贝壳数据（成交价、带看量、关注度）是最有参考价值的

### 5. 成交记录分析

访问成交页面，提取近期成交：
- 成交日期、面积、总价、单价
- 对比挂牌价 vs 成交价（议价空间）
- 成交周期趋势

### 6. 输出报告

发送到飞书，格式包含：

```
🏠 <小区名> 房产调研报告

📊 基本信息
• 参考均价: XX元/㎡
• 位置: XX区XX街道
• 建成年代: XXXX年
• 物业费: XX元/㎡/月
• 总户数: XX户

🔑 在售房源（按户型分组）
【两室】XX-XX万（XX-XX元/㎡）
• 具体房源列表...

【三室】XX-XX万（XX-XX元/㎡）
• 具体房源列表...

📈 价格分析
• 单价范围: XX ~ XX 元/㎡
• 主流价格: XX ~ XX 元/㎡
• 议价空间参考: XX%

⚠️ 注意事项
• 数据来源及时效说明
```

## 操作注意事项

- **不要 screenshot**，只用 snapshot 获取页面结构
- 页面加载等2秒再 snapshot
- 房天下分页：在售房源通常有多页，URL 中 `i3<页码>` 控制翻页
- 大型小区（如天景山公寓）包含多个苑，注意区分
- 安置房小区可能没有单独的苑级别数据
- 均价仅供参考，实际成交价可能有较大偏差
